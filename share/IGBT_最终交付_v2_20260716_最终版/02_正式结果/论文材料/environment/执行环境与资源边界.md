# 执行环境与资源边界

## 可观察机制

- `scripts/run_igbt_seb_case.ps1` 定义隔离的 `CaseId/AttemptId`、`ExecutionMode`、输入 SHA-256、私有产物 manifest 和运行片段；允许模式为 `PARALLEL_EXPLORATORY`、`SERIAL_CONFIRMATORY`、`DIAGNOSTIC_CLOSURE`。
- `config/sentaurus-core-policy.json` 指定：保留 core 0、最多 4 个受管槽位、租约根、20 s 锁超时、未管理进程排除 affinity。
- `scripts/sdevice_core_lease.sh` 与 `team_setup/skills/sentaurus-vm-runner/references/core-scheduling.md` 共同定义 Linux VM 上的原子单核租约；`codex-workflows`/`sentaurus-vm-runner` 负责把执行、资源和产物审计接到同一 run 记录。
- 包装器的 `run_provenance.csv` 设计字段包括 `allocation_mode`、`cpu_core`、`sdevice_threads`、`lease_acquired`、`lease_released`、`affinity_verification`、`exit_code` 和 `wall_time_seconds`。
- 7/11 记录显示，特定半步网格 1/2/4 线程等预算探针均只到 0.3125 V，2/4 线程 CPU 更高而无进度收益；该证据支持该条件下选择单线程，不能泛化为线程数的普适规律。

## 合规证据的边界

本包没有逐运行公开租约日志，也没有完整 run registry：`run_registry.csv` 只含一条 `legacy` 诊断闭合行。因此**不能**计算租约获取/释放率、CPU affinity 合规率、实际并发数、全体 run wall-time 分布或 7/14 的逐案运行资源统计。论文只能写“工作流要求并记录这些字段；公开账本不足以验证全体运行合规”。

## 人工与自动化分工

研究者冻结目标、模型边界、允许调优和停止规则；在 7/11 记录中，纠错性人工参数干预为 0，另有 1 次用户批准的范围决策。Codex 在既定范围内完成命令编排、日志/产物检查、有限变体、门控和失败保留。研究者仍负责策略批准、物理解释、版权/隐私决定和论文结论。

`token`/`tool-call` 计数、人工基线、重复运行和跨模型/跨设备对照都不可用，不能作为效率或优越性指标。

## 证据缺口声明

本材料没有人工基线、完整 token/tool-call 账本、重复实验或完整逐 run lease/affinity 账本；不得用代理自身输出构造“人工参考”或效率分母。所有 7/14 数字来自正式 sidecar/验收表，所有低 LET 与 650 V 记录都保留其附录或 `PENDING` 状态。
