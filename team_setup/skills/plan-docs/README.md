# plan-docs

> 一个用于「项目规划与文档化」的 Claude Code / Codex 技能（Skill）。
> 把模糊的人类想法，收敛成一套**约束 AI、细节化到按钮级**的文档体系，再据此生成原型、开发追踪与可直接执行的任务提示词。

## 为什么做这个

和 AI 一起开发项目时，大多数人只聊**大方向**，不梳理细节。于是 AI 缺乏约束、缺乏精确依据，开始**自由发挥**——做着做着就跑偏了（上下文越长越明显）。

`plan-docs` 解决两件事：

1. **约束 AI** —— 给 AI 套上明确的边界与依据，让它只在被界定的范围内行动。
2. **细节化功能开发** —— 把需求与实现拆到足够细的颗粒度（细到「点这个按钮的逻辑」、细到「一条 debug 语句」），让每一步都有清晰、可执行的定义。

底层判断：**人和 AI 有「代沟」**。人用原始口语表达，AI 理解有偏差。这个技能的工作，就是不断把「人话」翻译成「AI 能精确执行、可自检」的结构化文档与提示词。

## 五大功能

| # | 功能 | 做什么 |
|---|------|--------|
| 一 | **原话留存** | 全程逐字记录用户每句话，不改写不总结，需求可溯源 |
| 二 | **PM 视角需求讨论 + 分层文档** | 以产品经理视角讨论实用性，产出从「项目说明书 → 需求 → 架构 → 索引 → 无限递归模块树」的文档网络 |
| 三 | **高保真原型** | 据微观文档按需生成 HTML/CSS 可交互原型 |
| 四 | **开发追踪 + 顺序梳理** | 依赖推导开发先后顺序 + 状态看板，每条链接到对应文档 |
| 五 | **任务微观化 + AI 友好提示词** | 拆到极细的任务，生成「给 AI 的源数据」提示词；支持直接复制发送，也支持 `/goal` 自动逐个执行 |

## 核心机制：防漂移的文档网络

不是一堆散落的碎文件，而是**有索引、前后逻辑通顺、互相关联**的网络（"C 方案"：中心索引 + 文档内上下游）：

- **纵向（包含/父子）**：模块 → 子模块 → 子子模块……可无限递归，这棵树本身就是开发关联关系。
- **横向（依赖/调用）**：跨模块的触发/调用（如「按钮X」触发「模块B 的某流程」）。
- **中心索引** `03-索引.md` 是**单一事实来源**，改结构先改它。
- **每个文档头部**统一标注 `上级 / 下级 / 依赖`，逐层可追。

技能在**目标项目**根下生成并维护 `docs/`：

```
docs/
├── 用户原话.md            # 功能一：原始输入，溯源用
├── 00-项目说明书.md       # 项目是什么、为谁做、目标与边界
├── 01-需求文档.md         # 功能清单、优先级、用户场景
├── 02-架构文档.md         # 技术栈、模块划分、数据流
├── 03-索引.md             # 全局模块树 + 横向依赖总表【单一事实来源】
├── 04-开发追踪.md         # 开发顺序 + 状态看板
├── 05-术语表.md
├── 06-数据字典.md
├── 07-API文档.md
├── 08-测试用例.md
└── modules/               # 功能二：微观实现文档（可无限递归）
    └── 模块A/
        ├── _A.md
        ├── 子模块A1/
        │   ├── _A1.md
        │   └── 按钮X.md   # 最小叶子文档：场景 / 触发 / 逻辑 / 上级 / 依赖
        └── ...
```

## 两种入口模式

- **新项目 → 引导式**：从零按 `原话 → 需求&文档 → 原型 → 开发追踪 → 任务拆分` 走全程。
- **老项目 → 初始化式**：AI 先读懂现有代码，**逆向构建**出文档树，帮你更了解自己的项目，再继续完善。

各功能也支持**单独调用**（如「更新开发追踪」「新增一个模块文档」「为某任务生成提示词」），无需重走全程。

## 安装

### Claude Code

把本仓库放到 Claude Code 的技能目录下：

```bash
# 用户级（全局可用）
git clone https://github.com/2270525352/plan-docs.git ~/.claude/skills/plan-docs

# 或项目级（仅当前项目）
git clone https://github.com/2270525352/plan-docs.git .claude/skills/plan-docs
```

随后在 Claude Code 里，当你说「帮我规划这个项目 / 梳理需求 / 建文档树」时，技能会被自动触发；也可显式调用 `/plan-docs`。

## 使用

直接用自然语言描述你的项目想法即可，例如：

> 我想做一个 xxx，帮我规划一下。

技能会：
1. 开始**逐字记录**你的原话；
2. 以 **PM 视角**追问目标、用户、场景、实用性；
3. 自顶向下建立分层文档，并细化到模块/按钮级；
4. 按需生成**高保真原型**；
5. 推导**开发顺序**并建立追踪看板；
6. 在你确认文档后，把任务**拆到极细**并生成可执行的**提示词**。

## 关于 `/goal`（功能五·可选）

`/goal` 是 Claude Code 与 OpenAI Codex CLI 内置的「目标驱动自主执行」模式：给一个**可验证的完成条件**，AI 自主多轮执行，由独立检查模型逐轮校验，达成即停。

功能五生成的提示词：
- **方式 A（基础）**：直接复制发给 AI，手动驱动逐个任务。
- **方式 B（可选）**：写成带可验证完成条件的形式，交给 `/goal` 自动逐个跑。

`/goal` **不是必须**——「直接复制发送」始终是基础形态。

```
plan-docs/
├── SKILL.md            # 技能主入口：触发描述、调度逻辑、核心纪律
├── references/         # 各功能 / 模式的详细 SOP
│   ├── 功能1-原话留存.md
│   ├── 功能2-需求讨论与分层文档.md
│   ├── 功能3-高保真原型.md
│   ├── 功能4-开发追踪与顺序.md
│   ├── 功能5-任务微观化与提示词.md
│   ├── 模式-新项目引导.md
│   └── 模式-老项目初始化.md
├── templates/          # 目标项目要生成的文档模板
└── docs/               # 本技能自身的设计产物（原话留存 + 设计文档）
```

> `docs/` 里保留了这个技能**自身被设计出来的全过程**——用户原话与设计文档。它本身就是 `plan-docs` 方法论的一次实践示例。
