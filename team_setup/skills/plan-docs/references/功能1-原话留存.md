# 功能一 · 原话留存 SOP

## 目的
忠实、逐字保留用户说的每一句话，作为需求溯源的基准。当后续文档与用户本意出现偏差时，能回到原始表述对齐。

## 何时执行
**贯穿全程，持续执行。** 不是一次性动作——只要用户在对话中表达了需求、做了决定、提出了修正，就追加记录。

## 操作步骤
1. 先判断本次是否是**新需求**：
   - 新需求：在 `docs/changes/YYYY-MM-DD-需求短名/用户原话.md` 记录。
   - 初始化/旧文档维护：沿用目标项目现有 `docs/用户原话.md`。
2. 对 2026-07-02 之后的新需求，默认新建 `docs/changes/YYYY-MM-DD-需求短名/`，不要继续把所有原话追加进根 `docs/用户原话.md`。
3. 每当用户发言含**需求 / 决定 / 修正 / 偏好**时，把原话**逐字**追加进去。
4. 按日期分段；同一天多条按顺序排列。
5. 给关键发言加简短括注标签，便于回查，例如：`（修正功能五）`、`（回答 AI 提问·原型形态）`。

## 纪律
- **不改写、不总结、不润色**——保留错别字和口语，原貌即证据。
- **区分来源**：AI 自己的查证结论、推断、设计建议**不能**混进用户原话；如确需记录，必须显式标注"（AI 查证结论，非用户原话）"。
- 用户做的**选择题答案**也要记录（连同被选中的选项文字）。

## 关联
- 是功能二需求文档的**上游来源**：需求文档中的每个关键判断，理想情况下能在原话中找到出处。
