# 模式 · 新项目引导式 SOP

## 适用
目标项目是**全新的**（没有或几乎没有代码、文档）。从零开始，按完整流程走一遍。

## 判断是不是新项目
- 目录基本为空，或只有脚手架/README。
- 没有 `docs/` 文档体系。
- 用户描述的是"我想做一个……"而非"我这个项目……"。

## 引导流程（按序）
1. **建文档骨架**：在项目根创建 `docs/`，先放入 `用户原话.md`。
2. **功能一·原话留存**：开场即开始记录，并贯穿全程。
3. **功能二·需求讨论 + 分层文档**：
   - 以 PM 视角追问目标/用户/场景/实用性。
   - 自顶向下：项目说明书 → 需求文档 → 架构文档 → 索引 → 递归模块树。
   - 每完成一层与用户确认。
4. **功能三·高保真原型**：微观文档就绪后，**询问**用户是否要原型；要则生成 HTML/CSS。
5. **功能四·开发追踪 + 顺序**：据索引依赖推导开发顺序，建状态看板。
6. **功能五·任务微观化 + 提示词**：**等用户完全确认文档后**，逐模块拆任务、生成提示词。

## 节奏纪律
- 每一步产出后，**停下来与用户确认**再进入下一步，避免一路狂奔跑偏。
- 全程持续记录原话。
- 用户可随时打断、跳步或要求单独调用某功能——尊重用户节奏，不强制串行。
