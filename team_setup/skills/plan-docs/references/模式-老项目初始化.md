# 模式 · 老项目初始化式 SOP

## 适用
目标项目**已有代码/结构**，但缺少（或不完整）本技能的文档体系。目的：让 AI 先**了解现有项目**，**逆向构建**出文档树，帮用户更了解自己的项目，再去完善与继续开发。

## 判断是不是老项目
- 已有实质代码、目录结构、依赖配置。
- 用户描述为"我这个项目……""帮我梳理现有的……"。

## 逆向构建流程
1. **扫描与理解**（只读，先不改代码）：
   - 读 README、package/依赖清单、目录结构、入口文件、配置。
   - 识别技术栈、主要模块边界、对外接口、数据模型。
   - 必要时让用户补充：项目背景、目标用户、哪些是核心功能。
2. **建文档骨架**：创建 `docs/`，放入 `用户原话.md`（记录用户对项目的口述背景）。
3. **逆向回填顶层文档**：
   - `00-项目说明书.md`：根据代码 + 用户口述，归纳项目是什么、为谁做。**推断的内容要标注"待用户确认"。**
   - `02-架构文档.md`：依据真实代码结构画出现有架构、技术栈、模块划分。
   - `01-需求文档.md`：从已实现功能反推现有需求清单（标注哪些是推断）。
4. **建索引与模块树**：`03-索引.md` 按代码现状画模块树 + 横向依赖；`modules/` 下为每个真实模块建 `_X.md`，逐步细化到关键交互。
5. **校对**：与用户逐项确认推断是否符合事实，修正偏差。
6. **进入完善/开发**：文档树建好后，转入正常流程——补缺失细节、规划新功能（功能二）、追踪开发（功能四）、拆任务（功能五）。

## 纪律
- **先理解，后下笔**：逆向文档必须基于真实代码，不臆造。
- **明确区分"现状"与"推断"**：凡是 AI 推断而非代码明证的，标注待确认。
- **不擅自改代码**：本模式只建文档；改代码走功能五的任务流程，且需用户确认。

## 关联
- 完成后衔接到功能二/三/四/五，与新项目流程合流。
